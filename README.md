## Результат

```
Test Top-1 Accuracy: 0.2990
Test Top-3 Accuracy: 0.5298
```
Модель угадывает правильный ход в 30% случаев и попадает в top3 в половине случаем. Тут надо учесть несколько моментов:  
1.  Игроки могут делать субоптимальные, но стратегически верные ходы
2. Игроки с разным уровенем и стилем игры, из-за чего в данных много шума
### Немного о решении
Мое решение -- это легкий resnet  
Поработал с данными, нашел невозможный ход в одной строке - удалил. Далее подготовил обработку данных, чтобы все было представимо в виде тензоров.
Создал все необходимые классы для обучения. Пробовал изначально 4 блока resnet без допаута - модель слишком быстро переобучалась. Уменьшил модель, добавил дропаут.

### Метрики
Top1, Top3. Top1 - accuracy, стандартная метрика для задачи классификации. Но для шахмат это не столь важно угадывать точь-в-точь ход. Частно локально неоптимальный ход глобально имеет много смысла. Можно пожертвовать ферзя и на следующих ход объявить мат. Потому больше смысла имеет смотреть на top3, top5 метрики.
Я поставил себе небольшую цель - пусть модель научится решать детский мат. Не научилась, но хочу отметить, что ход который она делает имеет под собой какой-то смысл. Вместо ферзь f7 в ответ на коня f6 она ходит ферзь f3. Это стандартный ответ на пешку g6. То есть модель что-то понимает. Так что я считаю, что большее количество данных с моим подходом привело бы к лучшим метрикам.

### Как решать задачу для отдельного человека?  
Добавить embedding игрока на новую фичу. Таким образом модель будет обучаться учитываться поведение игрока: агрессивный стиль, склонность в жертве, уровень скилла и тп
Обучить модель на всем датасете -> дообучить для отдельного игрока.
